# 角色与目标
你是一名世界顶尖的前端开发工程师、UI/UX 设计师和 AI 产品架构师。你的任务是使用 React 构建一个高保真、可交互的“AI 行动学习催化师”移动端 Web 完整原型。

# 技术栈与全局规范
- **框架**：React (推荐使用 Vite 搭配 React Router，或 Next.js App Router)
- **样式**：Tailwind CSS，严格遵循 **移动端优先 (Mobile-First)**，最大宽度 `max-width: 480px`，桌面端居中，外围浅灰背景。整体风格参考企业级 SaaS 与高级微信小程序（留白充足、排版优雅）。
- **组件库与图标**：使用 `lucide-react` 作为图标库，UI 组件（按钮、卡片、输入框）需具备现代感（圆角 `rounded-xl` 或 `rounded-2xl`，细腻的阴影 `shadow-sm`）。
- **数据可视化**：使用 `recharts` 绘制雷达图。
- **动画**：使用 `framer-motion` 处理页面切换、AI 思考加载态、卡片弹出的动效。
- **AI 视觉语言**：所有 AI 介入的元素（如评分、洞察、生成的提问），请统一使用带有科技感的紫/蓝渐变色调、发光边框或 Sparkles 图标，以区分人工输入和 AI 输出。

# 项目核心背景
这是一款帮助企业团队进行“行动学习”的智能小程序。它用 AI Agent 替代部分“催化师”角色。核心业务流转分为四个核心阶段，请你通过顶部的进度条（Step 1 到 Step 4）或路由来实现页面切换。

# 核心页面与功能拆解（需全部实现并串联）：

## 阶段 1：课题提报与 AI 预审 (对应图纸：选题阶段)
**目标**：拦截质量差的课题，AI 自动打分并给出优化建议。
- **UI 元素**：
  - 表单区域：课题名称、背景现状、核心痛点、已尝试行动。
  - AI 质量评估卡片（点击提交后展示动画并浮现）：
    - 圆形进度条展示总分（例如 6/10 分）。
    - 6个维度的雷达图或条形图展示得分（聚焦性、结果导向、单一议题、未知性、可控性、学习性）。
    - ⚠️ **优化建议模块**：“课题过于宽泛，建议聚焦‘某某具体场景’，而非泛泛而谈。”
    - **同类优质示例推荐**（3个）。
  - 操作区：“确认修改” 与 “提交为最终课题”。

## 阶段 2：研讨开场与 AI 事前验尸 (对应图纸：事前诊断阶段)
**目标**：在提问风暴开始前，给案主和团队敲响警钟。
- **UI 元素**：
  - 当前确定课题的大卡片展示。
  - 🚨 **AI 事前验尸预警（高亮设计）**：深色或红色警示卡片。文案如：“基于历史数据分析，若延续当前策略，预测未来一个月客诉量可能上升20%，且员工流失风险增加。建议本次研讨重点关注‘信任重建’。”
  - “开始提问风暴”的巨大引导按钮。

## 阶段 3：提问风暴与盲区诊断 (对应图纸：5F 提问与雷达图阶段)
**目标**：小组成员提问，AI 实时归类并发现思维盲区。
- **UI 元素**：
  - **核心图表区**：使用 `recharts` 渲染 5F 维度雷达图（事实、感受、行动、分析、聚焦）。设定 Mock 数据让“感受类”占比为 0%。
  - **AI 动态预警**：“检测到‘感受类’提问严重缺失！”
  - **影子提问生成区**：自动弹出 2-3 个 AI 补充的感受类提问（如“客户看到亏损时的第一反应是什么？”），并带有“👍 采用” 和 “❌ 未采用” 按钮。采用后更新雷达图数据。
  - **人工提问流**：瀑布流展示其他成员的问题及对应的 AI 标签（如 `[事实类]`）。
  - **底部悬浮输入框**：支持实时输入，占位符为“输入你的提问... AI 将实时校验”。

## 阶段 4：认知重构与行动计划 (对应图纸：研讨收尾与 KPI 阶段)
**目标**：收敛黄金问题，制定落地计划。
- **UI 元素**：
  - **被选出的黄金问题展示**（突出显示）。
  - **四层次重构区**：简单的文本域供用户输入反思。
  - **KPI / 行动计划制定表单**：
    - 列表视图，每行包含：责任人 (Owner)、具体行动任务 (Action)、截止日期 (Deadline)。
    - 提供“新增一行”功能。
  - **最终总结报告**：一个带有“生成总结报告”按钮的区域，点击后展示一个漂亮的总结卡片。

# 交互与数据流要求
1. **全局 State**：请在最外层维护一个全局状态（如使用 Context 或简单的顶层 State），存储 `currentTopic`（当前课题）、`questions`（提问列表）、`radarData`（雷达图数据）。这样当阶段 1 确定的课题可以在阶段 2 和 3 中显示。
2. **Mock 数据**：必须填充逼真的中文 Mock 数据（如“如何通过流程优化提升销售团队的客户跟进效率”），不要让页面显得空旷。
3. **流畅体验**：阶段之间的切换请使用 `framer-motion` 添加左右滑动的平滑过渡效果。
